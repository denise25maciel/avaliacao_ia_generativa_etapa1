Develop a high-fidelity interface proposal, limited to Streamlit components, for the problem and solution described below:

Identified Problem – Professor Ana currently experiences the following difficulties:

She needs to manually search for competitive exam exercises to use them in tests.

She needs to adapt exercises into simpler ones or link them to SDG (Sustainable Development Goals) themes.

She needs to determine the difficulty level, that is, whether each exercise is simple, intermediate, or advanced.

Based on the selected questions, she needs to organize them by topic and progressively display the questions according to difficulty.

Because the process is carried out manually, Ana’s main pain points are the high amount of time required to manually develop the questions and the lack of version control for the original questions.

Proposed Solution: Development of a web-based platform. The platform allows:

Uploading a database of exercises

Registering exercises from the uploaded data

Structured search with multiple filters

Adapting an exercise so that multiple simpler exercises can be generated from it, with version history

Adapting an exercise so that it addresses an SDG theme


I've designed the Exercise Management Dashboard for Professor Ana.

This interface follows a Streamlit-inspired layout to provide a familiar and efficient workspace:

Sidebar Controls: Easy access to file uploads for exercise databases and granular filters for Subject, Difficulty, and SDG themes.
Actionable Insights: Top-level metrics show the current state of her database at a glance.
Search & Manage: A centralized data table allows her to search, sort, and identify which questions need adaptation or SDG alignment.
How does this initial dashboard look? We can proceed to design the Exercise Adaptation flow or the Version History view next.
